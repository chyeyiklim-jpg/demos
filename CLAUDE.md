# FinanceTrack — Claude Code Guide

## Project Overview

Investment portfolio tracker for stocks and crypto. Built with Next.js 14 (App Router), TypeScript, and a multi-agent AI architecture powered by the Claude API.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 14 App Router |
| Language | TypeScript (strict) |
| Database | libSQL (local-first) + Drizzle ORM |
| Cache | Redis (ioredis) |
| AI | Anthropic SDK (`claude-sonnet-4-6`) |
| Styling | Tailwind CSS |
| Testing | Vitest (serial, `singleFork`) |

## Common Commands

```bash
npm run dev          # start dev server at localhost:3000
npm run build        # production build
npm test             # run all tests once
npx vitest           # run tests in watch mode
npx drizzle-kit generate   # generate migration after schema change
npx drizzle-kit migrate    # apply migrations to local.db
```

## Architecture — Agent Rules

1. **Only Orchestrator imports other agents.** Specialist agents never import each other.
2. **All agents return `AgentResult<T>`** — never throw across agent boundaries.
3. **Specialists are stateless** — all persistence goes through `db/client.ts` or `lib/cache/redis.ts`.
4. **Use `ok()` and `fail()` helpers** from `src/types/agents.ts` — don't construct the union manually.

```typescript
// correct
return ok(data)
return fail('message', 'DB_ERROR')

// wrong
return { success: true, data }
```

## Database

Schema is in `src/db/schema.ts`. After any schema change:

```bash
npx drizzle-kit generate
npx drizzle-kit migrate
```

Local DB file: `local.db` (gitignored). VPS sync configured via `LIBSQL_SYNC_URL` in `.env.local`.

## Environment Variables

Copy `.env.local` and fill in real values:

```
LIBSQL_URL=file:local.db
LIBSQL_SYNC_URL=http://<VPS_IP>:8080   # Tencent Lighthouse sqld
LIBSQL_AUTH_TOKEN=
REDIS_URL=redis://localhost:6379
ANTHROPIC_API_KEY=sk-ant-...
YAHOO_FINANCE_BASE_URL=https://query1.finance.yahoo.com
```

## Testing

Tests use a **single shared SQLite file** — they must run serially (`singleFork: true` in vitest.config.ts). Never change the pool config or tests will get `SQLITE_BUSY` errors.

Each test file that writes to the DB must clean up in `beforeEach`:

```typescript
beforeEach(async () => {
  await db.delete(trades)
  await db.delete(positions)
})
```
